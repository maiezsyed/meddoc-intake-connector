Meddoc Intake Connector (Preview)

Athenahealth Sandbox • System-to-system (2‑legged OAuth)

Overview

Meddoc Intake Connector centralizes patient intake documents and makes them searchable. Phase‑1 integrates with the Athenahealth Preview environment to associate documents with sandbox patient identifiers while keeping files securely in Azure storage. No end‑user login is required.

  • Upload: Accept files from automations (e.g., Logic Apps)
  • Index: Cosmos DB metadata + Azure AI Search text snippets
  • Retrieve: Short‑lived SAS links for secure downloads
  • Link (optional): Store Athena patient/encounter IDs for future write‑back

Key Facts

  • Not ONC‑Certified API (CAPI)
  • Not a PHR / no patient login
  • Sandbox only (non‑production)

Security & Privacy

  • OAuth2 client‑credentials (backend‑only)
  • TLS/HTTPS in transit; Azure‑encrypted at rest
  • Short‑lived download links (SAS)
  • Intended for sandbox test data. Do not upload real PHI without appropriate agreements and production hardening.

Getting Started

  1. Request a demo or share sandbox patient identifiers.
  2. We’ll provide API base URL and token scope.
  3. Use our Postman collection to POST /documents, GET /search, and GET /documents/{id}.
